Learned answers

Whenever you manually answer a question the extension didn't recognize, it remembers your answer here (matched by question wording) and reuses it next time. Edit or delete entries below. Questions like "Have you applied to this company before?" are deliberately never remembered here - those are always asked fresh.

You can also teach it an answer directly - useful for fields that don't save automatically (e.g. a "State" field that's a text box with its own dropdown instead of a real <select>). Copy the question/label text as it appears on the page (or close to it - matching is fuzzy) and the answer you want used.

Regex learned answers (advanced)

For multiple-choice / radio questions where the wording varies a lot but you always want the same kind of answer - e.g. "Were you referred by a current employee?" with options Yes/No - you can match the question with a regular expression and pick the option whose own text matches an answer regular expression.

Both patterns use JavaScript regex syntax (not PCRE - inline flags like (?i) aren't supported; use the Flags field instead, default i for case-insensitive). The question pattern is tested against the field's full label/question text (unanchored, so referred|current matches anywhere in "Were you referred by a current employee?"). The answer pattern is tested against each option's own text (e.g. \bno\b matches an option labeled "No"). Only applies to radio/checkbox groups and <select> dropdowns - there's no single value to type into a text field. The same "never reused across companies" guardrail (e.g. "have you applied here before") still applies.
